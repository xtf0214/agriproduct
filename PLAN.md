# 农产品电商系统实施计划（按当前项目与 API.md 对齐）

> 更新时间：2026-02-21
> 说明：本计划基于当前仓库实际代码结构与 `API.md` 现状整理，替换旧版“多模块微服务”规划。

## 1. 当前项目现状

## 1.1 目录与形态

- 当前后端为 **单体 Spring Boot 项目**：`agriproduct-backend/`
- Web 前端目录：`agriproduct-web/`
  - `agriproduct-web-merchant/`：已有完整页面与路由
  - `agriproduct-web-admin/`：目录存在，但当前为空
- 小程序目录：`agriproduct-miniprogram/`（当前为空）
- 数据库脚本：`sql/agriproduct.sql`
- 接口文档：`API.md`

## 1.2 技术栈（按代码）

- Java 17
- Spring Boot 3.2.0
- MyBatis-Plus 3.5.5
- MySQL + Druid
- Redis
- JWT（jjwt）
- OpenAPI/Knife4j
- 商家端前端：Vue3 + Vite + TypeScript + Pinia

| 风险 | 影响 | 处理策略 |
